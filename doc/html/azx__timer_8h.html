Detailed Description

A better way to use timers.

Version
1.0.2
Dependencies
core/azx_log core/azx_tasks core/azx_utils
Author
Sorin Basca
Date
10/02/2019

This library provides a convenient way to define and use timers in the application.

One can use asynchronous timer with azx_timer_init() to receive notification on an existing task (see azx_tasks.h), or specify a custom callback using azx_timer_initWithCb(). Timers can then be used with azx_timer_start().

Furthermore, timestamp related operations can be performed. For example you can make a cool-down timer synchronously using azx_timer_getTimestampFromNow() and the related functions.

◆ azx_expiration_cb

typedef void(* azx_expiration_cb) (void *ctx, AZX_TIMER_ID timer_id)

Callback issued when a timer expires.

The callback will be issued on a dedicate task, therefore the handler can do any intense/blocking processing it requires. The only limitation is that other timers using the callback will be pending until the handling is done, but the real-time tasks will not be affected.

Parameters
[in]ctxThe context that was passed to azx_timer_initWithCb()
[in]timer_idThe ID of the timer as returned by azx_timer_initWithCb()

◆ azx_timer_init()

AZX_TIMER_ID azx_timer_init ( INT32  task_id,
INT32  type,
UINT32  duration_ms 
)

Initializes a timer.

The timer can be started for a certain amount of time. Once it expires a message will be queued to the task provided. Do not expect that the task will handle the expired notification at the exact time, as there may be some delays introduced in handling the queued message. The only guarantee is that at least the minimum duration has passed.

The message sent on expiry will have the type as defined here, param1 will be the ID of the timer and param2 will be unused.

Parameters
[in]task_idThe task where the expired message is stored.
[in]typeThe type of the message to notify as the expiry
[in]duration_msThe interval between callbacks. If it doesn't change, set it to something, otherwise leave it 0 and use the parameter in azx_timer_start().
Returns
The ID of the created timer, or NO_AZX_TIMER_ID if something failed.

AZX_TIMER_ID azx_timer_initWithCb ( azx_expiration_cb  cb,
void *  ctx,
UINT32  duration_ms 
)

Initializes a timer that executes a cb upon completion.

The timer can be started for a certain amount of time. Once it expires the callback function will be executed.

Warning
This is running on the timer task. Keep the processing to the min (queue to other tasks)

Do not expect that the task will handle the expired notification at the exact time, as there may be some delays introduced in handling the queued message. The only guarantee is that at least the minimum duration has passed.

The message sent on expiry will have the type as defined here, param1 will be the ID of the timer and param2 will be unused.

Parameters
[in]cbThe function to call upon timer expiration
[in]ctxPasses this to the cb function
[in]duration_msThe interval between callbacks. If it doesn't change, set it to something, otherwise leave it 0 and use the parameter in azx_timer_start().
Returns
The ID of the created timer, or NO_AZX_TIMER_ID if something failed.
