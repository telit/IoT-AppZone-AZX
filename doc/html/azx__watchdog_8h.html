Detailed Description

Software watchdog to detects stalling tasks.

Dependencies

Version

Author
Ioannis Demetriou
Sorin Basca
Date
10/02/2019

You can use this watchdog to detect in software if the application is stalling in some task. This is useful if some task processes something that doesn't end.

Just call azx_start_watchdog() to start it and signal to it that everything is OK when checkpoints are reached by calling azx_signal_watchdog().

To adjust he watchdog timeout, modify AZX_WATCHDOG_TIMEOUT_MS.

◆ AZX_WATCHDOG_TIMEOUT_MS

#define AZX_WATCHDOG_TIMEOUT_MS   40000

The maximum value between two signals.

If the interval in milliseconds elapses without a call to azx_signal_watchdog(), then the modem will reset.

void azx_start_watchdog ( void  )

Starts the watchdog to monitor for any stalling threads.

This should be called only once. Once the watchdog is started it does not stop, unless a reset happens.
